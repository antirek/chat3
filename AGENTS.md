# Правила разработки проекта Chat3

используй русский язык

1. **Соблюдайте пространства имён сообщений** — синхронизируйте валидацию `internal.*`, `system.*` и `user.*` в схемах, контроллерах, моделях, документах и ​​начальных данных при каждом изменении типов сообщений.
2. **Используйте скрипты проекта** — выполняйте задачи обслуживания с помощью предоставленных скриптов npm (`npm run seed`, `npm run generate-key`, `npm run start:tenant-api`, `npm run start:all` и т. д.) вместо произвольных команд.
3. **Обновите учётные данные демонстрационного API** — после генерации нового ключа перед передачей работы распространите его на каждую страницу `src/apps/api-test/public/api-test*.html`. 
4. **Перезапуск через вспомогательный скрипт** — после повторного заполнения данных или ротации ключей перезапустите службы с помощью `/home/sergey/Projects/tmp3/chat3/restart.sh`, чтобы сервер API и рабочий процесс корректно приняли изменения.
5. **Зеркалирование тестовой инфраструктуры** — используйте `mongodb-memory-server` и `@onify/fake-amqplib` для интеграционных тестов контроллера/утилиты, чтобы соответствовать принятому в проекте подходу к тестированию.
6. коммит только по команде
